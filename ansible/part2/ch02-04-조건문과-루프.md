# Part 2: 조건문과 루프

## 4.1 조건문 (when)

**when**: 조건이 참(True)일 때만 태스크 실행

### 주요 사용 사례
- OS 종류에 따라 다른 패키지 설치
- 변수 값에 따라 서비스 설정
- 파일/디렉터리 존재 여부 확인 후 작업
- 이전 태스크 결과에 따라 실행

### 기본 문법
```yaml
- name: 조건부 태스크
  command: some_command
  when: <condition>
```

---

### 예제 1: Ansible Facts 조건
```yaml
- name: OS별 Apache 설치
  hosts: webservers
  tasks:
    - name: RedHat에 httpd
      yum:
        name: httpd
        state: present
      when: ansible_facts['os_family'] == "RedHat"

    - name: Debian에 apache2
      apt:
        name: apache2
        state: present
      when: ansible_facts['os_family'] == "Debian"
```

---

### 예제 2: 변수 조건
```yaml
- name: 환경별 배포
  hosts: all
  become: yes
  vars:
    deploy_env: "production"

  tasks:
    - name: Production에서만 마이그레이션
      command: /opt/app/run_migrations.sh
      when: deploy_env == "production"

    - name: Staging 메시지
      debug:
        msg: "This is staging. No migrations."
      when: deploy_env == "staging"
```

---

### 예제 3: register 결과 조건
```yaml
- name: 파일 존재 확인 후 작업
  hosts: all
  tasks:
    - name: nginx.conf 존재 확인
      stat:
        path: /etc/nginx/nginx.conf
      register: file_status

    - name: 파일 내용 출력
      command: cat /etc/nginx/nginx.conf
      when: file_status.stat.exists
      register: file_content

    - name: 내용 표시
      debug:
        var: file_content.stdout
      when: file_status.stat.exists
```

---

### 예제 4: 여러 조건 결합 (and/or)
```yaml
# and: 리스트 형태로 나열
- name: Ubuntu 24에 설치
  apt:
    name: apache2
  when:
    - ansible_facts['distribution'] == "Ubuntu"
    - ansible_facts['distribution_major_version'] == "24"

# or: 명시적 표현
- name: CentOS 또는 RHEL에 설치
  yum:
    name: httpd
  when: ansible_facts['distribution'] == "CentOS" or ansible_facts['distribution'] == "RedHat"
```

## 4.2 루프 (loop)

**loop**: 태스크를 리스트의 각 항목에 대해 반복 실행  
**item**: 현재 처리 중인 항목 접근 변수

### 예제 1: 리스트 루프
```yaml
- name: 여러 패키지 설치
  hosts: all
  tasks:
    - name: 필수 도구 설치
      apt:
        name: "{{ item }}"
        state: present
      loop:
        - vim
        - git
        - curl
```

---

### 예제 2: 딕셔너리 리스트 루프
```yaml
- name: 여러 사용자 생성
  hosts: all
  become: yes
  tasks:
    - name: 사용자 추가
      user:
        name: "{{ item.name }}"
        comment: "{{ item.comment }}"
        groups: "{{ item.groups }}"
      loop:
        - name: alice
          comment: "Alice Smith"
          groups: developers
        - name: bob
          comment: "Bob Johnson"
          groups: users
```

---

## 4.3 조건문과 루프 결합

```yaml
- name: OS별 그룹에 사용자 추가
  hosts: all
  tasks:
    - name: devuser 추가
      user:
        name: devuser
        group: "{{ item }}"
      loop:
        - wheel
        - sudo
      when: >
        (ansible_facts['os_family'] == 'RedHat' and item == 'wheel') or
        (ansible_facts['os_family'] == 'Debian' and item == 'sudo')
```

## 작성 가이드

| 가이드 | 설명 |
|--------|------|
| 가독성 | 복잡한 when → vars에서 변수로 정의 |
| `loop` 사용 권장 | `with_*` 대신 `loop` 사용 |
| `loop_control` | `loop_var`로 변수명 변경 가능 |
| 테스트 | `--check` + `-v`로 확인 |

```yaml
# loop_var 예시
- name: 패키지 설치
  apt:
    name: "{{ package_name }}"
  loop:
    - vim
    - git
  loop_control:
    loop_var: package_name
```
