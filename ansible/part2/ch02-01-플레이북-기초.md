# Part 2: 플레이북 핵심 마스터하기

## 1. 플레이북 기초

### 1.1 YAML 문법 기초

**YAML (YAML Ain't Markup Language)**: 사람이 읽고 쓰기 쉬운 데이터 직렬화 형식

### 핵심 요소

| 요소 | 설명 |
|------|------|
| 들여쓰기 | 계층 구조를 표현 (스페이스 2칸 표준) |
| Key-Value | `키: 값` 형태로 데이터 정의 |
| 리스트 | `- 항목` 형태로 순차 나열 |

### 들여쓰기 규칙
```yaml
# 올바른 예시 (2칸 들여쓰기)
server:
  name: web01
  ip: 192.168.1.10
  ports:
    - 80
    - 443

# 잘못된 예시
server:
  name: web01
   ip: 192.168.1.10  # 들여쓰기 불일치 → 오류
```

### Key-Value 예시
```yaml
name: "My Web Server"
protocol: http
port: 8080
is_enabled: true
use_ssl: no
```

### 리스트 예시
```yaml
# 단순 리스트
packages:
  - vim
  - git
  - nginx

# Key-Value 쌍을 항목으로 가진 리스트 (Ansible tasks)
tasks:
  - name: Install Nginx
    apt:
      name: nginx
      state: present

  - name: Start Nginx
    service:
      name: nginx
      state: started
```

## 1.2 플레이북의 기본 구조

플레이북 = **Play(플레이)**의 목록, Play = 특정 호스트에 대한 Tasks 목록

### 전체 구조 예시
```yaml
---
# 첫 번째 플레이
- name: Configure web servers
  hosts: webservers
  become: true
  tasks:
    - name: Install nginx
      apt:
        name: nginx
        state: present

    - name: Start nginx service
      service:
        name: nginx
        state: started

# 두 번째 플레이
- name: Configure database servers
  hosts: dbservers
  become: true
  tasks:
    - name: Install mariadb
      apt:
        name: mariadb-server
        state: present
...
```

### 구성 요소 설명

| 요소 | 설명 |
|------|------|
| **Play** | 가장 큰 실행 단위 ("어떤 서버에 어떤 작업") |
| **name** | 플레이 설명 (선택, 가독성 향상) |
| **hosts** | 대상 서버 그룹/호스트 지정 |
| **become** | 권한 상승 (sudo) |
| **tasks** | 실행할 작업 목록 |

### hosts 값 종류
| 값 | 설명 |
|------|------|
| `webservers` | 인벤토리 그룹 |
| `all` | 모든 호스트 |
| `localhost` | 제어 노드 자신 |
| `192.168.1.20` | 특정 IP |
| `web*.example.com` | 패턴 |

### become 사용
```yaml
# 플레이 수준
- hosts: webservers
  become: true  # 모든 태스크에 적용

# 태스크 수준
tasks:
  - name: Install nginx
    apt:
      name: nginx
    become: true

  - name: Clone repository
    git:
      repo: https://github.com/example/project.git
    become: false  # 이 태스크만 권한 상승 안 함
```

## 1.3 ansible-playbook 명령어

**ansible-playbook**: 플레이북 실행 전용 명령어

### 기본 사용법
```bash
ansible-playbook [옵션] <플레이북.yml>
```

### 주요 옵션

#### 실행 대상 제어
| 옵션 | 설명 | 예시 |
|------|------|------|
| `-i` | 인벤토리 파일 지정 | `-i inventory.ini playbook.yml` |
| `--limit` | 특정 호스트/그룹만 실행 | `--limit webservers` |
| `-t` | 특정 태그만 실행 | `-t nginx` |
| `--skip-tags` | 특정 태그 건너뛰기 | `--skip-tags service` |

#### 연결 및 권한
| 옵션 | 설명 | 예시 |
|------|------|------|
| `-u, --user` | SSH 사용자 지정 | `-u kosa` |
| `-k, --ask-pass` | SSH 비밀번호 입력 | `-k` |
| `--become` | sudo 권한 사용 | `--become` |
| `-K` | sudo 비밀번호 입력 | `-K` |

#### 실행 모드 및 디버깅
| 옵션 | 설명 | 예시 |
|------|------|------|
| `-C, --check` | Dry Run (실제 변경 안 함) | `--check` |
| `-v, -vv, -vvv` | 상세 출력 | `-vv` |
| `-e` | 변수 전달 | `-e "ansible_user=kosa"` |

### 실전 예시
```bash
# 웹 서버에 Nginx 설치 태그만 Dry-Run
ansible-playbook deploy.yml --limit webservers -t nginx_install --check

# 특정 DB 서버에 다른 사용자로 접속
ansible-playbook db_setup.yml --limit db01 -u db_admin -k

# 변수 전달하여 특정 버전 설치
ansible-playbook install_app.yml -e "app_version=2.5.1"
```
