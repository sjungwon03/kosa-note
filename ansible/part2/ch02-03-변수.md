# Part 2: 변수 (Variables)

## 3.1 플레이북 내 변수 정의

### vars (직접 정의)
**장점**: 간편, 가독성 좋음  
**단점**: 재사용성 부족, 보안에 불리

```yaml
- name: vars 예제
  hosts: webservers
  become: yes
  vars:
    web_package: nginx
    web_service: nginx
    user_name: 'kosa'
    welcome_message: "Hello, {{ user_name }}!"

  tasks:
    - name: "{{ web_package }} 설치"
      package:
        name: "{{ web_package }}"
        state: present

    - name: 환영 메시지 생성
      copy:
        content: "<h1>{{ welcome_message }}</h1>"
        dest: /var/www/html/index.html
```

---

### vars_files (외부 파일)
**장점**: 재사용성, 모듈성, 보안 강화

**my_vars.yml**:
```yaml
web_package: nginx
web_service: nginx
user_name: 'kosa'
message:
  hello: "안녕하세요"
  user: "홍길동"
```

**플레이북**:
```yaml
- name: vars_files 예제
  hosts: webservers
  become: yes
  vars_files:
    - my_vars.yml

  tasks:
    - name: "{{ web_package }} 설치"
      package:
        name: "{{ web_package }}"

    - name: 환영 메시지
      copy:
        content: |
          <h1>{{ welcome_message }}</h1>
          <h2>{{ message.hello }}</h2>
          <h2>{{ message.user }}</h2>
        dest: /var/www/html/index.html
```

---

## 3.2 인벤토리에서 변수 정의

### 호스트 변수 (Host Variables)
**특정 호스트에만 적용** (우선순위 가장 높음)

#### 방법 A: 인벤토리 파일에 직접
```ini
[webservers]
prod-server ansible_host=192.168.40.175 server_env=Production
dev-server  ansible_host=192.168.40.176 server_env=Development
```

#### 방법 B: host_vars 디렉터리 (추천)
```
project/
├── inventory.ini
├── host_vars/
│   ├── prod-server.yml
│   └── dev-server.yml
```

**host_vars/prod-server.yml**:
```yaml
server_env: Production
```

---

### 그룹 변수 (Group Variables)
**그룹 전체에 공통 적용**

#### 방법 A: 인벤토리 파일 [그룹:vars]
```ini
[webservers:vars]
http_port=80
ntp_server=ntp.internal.net
```

#### 방법 B: group_vars 디렉터리 (추천)
```
project/
├── inventory.ini
├── group_vars/
│   ├── all.yml      # 모든 호스트
│   └── webservers.yml
```

**group_vars/webservers.yml**:
```yaml
http_port: 80
ntp_server: ntp.internal.net
```

**group_vars/all.yml**:
```yaml
ansible_user: kosa
ansible_python_interpreter: /usr/bin/python3.12
```

---

### 변수 우선순위
> 더 구체적인 변수가 덜 구체적인 변수를 덮어씀

| 우선순위 | 변수 위치 |
|----------|----------|
| 1 (最高) | 호스트 변수 |
| 2 | 그룹 변수 |
| 3 | all 그룹 변수 |

**예시**: `http_port: 8080` (host_vars) vs `http_port: 80` (group_vars) → 결과: **8080**

## 3.3 Ansible Facts

**Ansible Facts**: 관리 노드에서 자동 수집하는 시스템 정보

### Facts 수집
```yaml
- name: Facts 확인
  hosts: all
  tasks:
    - name: 모든 Facts 출력
      debug:
        var: ansible_facts
```

### 주요 Facts 변수
| 변수 | 설명 | 예시 |
|------|------|------|
| `ansible_facts['os_family']` | OS 계열 | RedHat, Debian |
| `ansible_facts['distribution']` | 배포판 | Ubuntu, CentOS |
| `ansible_facts['hostname']` | 호스트명 | web01 |
| `ansible_facts['default_ipv4']['address']` | IP 주소 | 192.168.1.123 |
| `ansible_facts['memtotal_mb']` | 메모리(MB) | 8024 |
| `ansible_facts['processor_vcpus']` | CPU 개수 | 4 |

---

### Facts 활용 예제

#### 예제 1: OS별 패키지 설치
```yaml
- name: OS에 따라 웹 서버 설치
  hosts: all
  become: yes
  tasks:
    - name: Debian/Ubuntu에 Apache
      package:
        name: apache2
      when: ansible_facts['os_family'] == "Debian"

    - name: RedHat/CentOS에 Apache
      package:
        name: httpd
      when: ansible_facts['os_family'] == "RedHat"
```

#### 예제 2: IP 주소로 방화벽 규칙
```yaml
- name: 현재 서버 IP 허용
  community.general.ufw:
    rule: allow
    src: "{{ ansible_facts['default_ipv4']['address'] }}"
```

#### 예제 3: 시스템 정보 보고서
**templates/report.j2**:
```
Hostname: {{ ansible_facts['hostname'] }}
OS: {{ ansible_facts['distribution'] }} {{ ansible_facts['distribution_version'] }}
CPU: {{ ansible_facts['processor_vcpus'] }}
Memory: {{ ansible_facts['memtotal_mb'] }} MB
IP: {{ ansible_facts['default_ipv4']['address'] }}
```

```yaml
- name: 보고서 생성
  hosts: all
  tasks:
    - name: 보고서 파일 생성
      template:
        src: templates/report.j2
        dest: /tmp/server_report.txt
```

---

### Facts 수집 제어
```yaml
# Facts 수집 안 함 (속도 향상)
- hosts: all
  gather_facts: no

# 필요한 정보만 수집
- hosts: all
  gather_subset:
    - network
    - min
```
