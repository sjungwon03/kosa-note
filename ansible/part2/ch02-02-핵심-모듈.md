# Part 2: 핵심 모듈 활용

## 2.1 파일 관리 모듈

### copy 모듈
**목적**: 컨트롤 노드의 파일을 원격 서버로 복사

| 파라미터 | 설명 |
|----------|------|
| `src` | 원본 파일 경로 (컨트롤 노드) |
| `dest` | 대상 파일 경로 (원격 서버) |
| `content` | 파일 내용 직접 지정 |
| `owner/group` | 소유자/그룹 설정 |
| `mode` | 권한 설정 (예: '0644') |
| `backup` | 덮어쓰기 전 백업 |

```yaml
- name: nginx.conf 복사
  ansible.builtin.copy:
    src: conf/nginx.conf
    dest: /etc/nginx/nginx.conf
    owner: root
    group: root
    mode: '0644'
    backup: yes

- name: info.txt 생성
  ansible.builtin.copy:
    content: "This is a simple text file."
    dest: /var/www/html/info.txt
    mode: '0644'
```

### template 모듈
**목적**: Jinja2 템플릿으로 동적 파일 생성

| 파라미터 | 설명 |
|----------|------|
| `src` | Jinja2 템플릿 파일 (.j2) |
| `dest` | 생성될 파일 경로 |
| `validate` | 파일 검증 명령어 (`%s`로 파일 경로 치환) |

**템플릿 파일 예시 (nginx.conf.j2)**:
```nginx
# Admin: {{ admin_email }}
server {
    listen {{ http_port | default(80) }};
    server_name {{ ansible_hostname }};
}
```

**플레이북 예시**:
```yaml
- name: Configure Nginx with template
  hosts: webservers
  become: true
  vars:
    http_port: 8080
    admin_email: admin@example.com
  tasks:
    - name: Create nginx.conf from template
      ansible.builtin.template:
        src: templates/nginx.conf.j2
        dest: /etc/nginx/nginx.conf
        owner: root
        mode: '0644'
      notify: Restart Nginx

  handlers:
    - name: Restart Nginx
      service:
        name: nginx
        state: restarted
```

### file 모듈
**목적**: 파일/디렉터리 상태 관리 (생성/삭제/권한)

| state 값 | 설명 |
|----------|------|
| `touch` | 빈 파일 생성 |
| `directory` | 디렉터리 생성 |
| `absent` | 삭제 |
| `link` | 심볼릭 링크 생성 |

```yaml
tasks:
  - name: 로그 디렉터리 생성
    file:
      path: /var/log/myapp
      state: directory
      mode: '0755'

  - name: 빈 로그파일 생성
    file:
      path: /var/log/myapp/app.log
      state: touch
      owner: kosa
      mode: '0644'

  - name: 심볼릭 링크 생성
    file:
      src: /var/log/myapp/app.log
      dest: ~/app.log.link
      state: link

  - name: 구성 파일 삭제
    file:
      path: /etc/old_config.conf
      state: absent
```

---

### lineinfile 모듈
**목적**: 파일의 특정 줄 관리 (추가/수정/삭제)

| 파라미터 | 설명 |
|----------|------|
| `path` | 파일 경로 |
| `line` | 존재해야 할 줄 내용 |
| `regexp` | 찾을 줄의 정규표현식 |
| `state` | present/absent |

```yaml
tasks:
  - name: hosts 파일에 엔트리 추가
    lineinfile:
      path: /etc/hosts
      line: "192.168.1.100 fileserver.example.com"

  - name: SELINUX 설정 변경
    lineinfile:
      path: /etc/selinux/config
      regexp: '^SELINUX='
      line: "SELINUX=disabled"

  - name: SSH 옵션 삭제
    lineinfile:
      path: /etc/ssh/sshd_config
      regexp: '^X11Forwarding'
      state: absent
```

## 2.2 패키지 관리 모듈

### apt 모듈 (Debian/Ubuntu)
| 파라미터 | 설명 |
|----------|------|
| `name` | 패키지 이름 |
| `state` | present/latest/absent |
| `update_cache` | apt-get update 실행 |

```yaml
- name: 패키지 업데이트 및 설치
  apt:
    update_cache: yes
    cache_valid_time: 3600

- name: nginx 설치
  apt:
    name: nginx
    state: present

- name: 여러 패키지 최신 버전 설치
  apt:
    name:
      - curl
      - git
    state: latest

- name: apache2 삭제
  apt:
    name: apache2
    state: absent
```

---

### yum/dnf 모듈 (RedHat/CentOS)
```yaml
- name: httpd, mariadb 설치
  dnf:
    name:
      - httpd
      - mariadb-server
    state: present
    update_cache: yes

- name: git 최신 버전 설치
  yum:
    name: git
    state: latest

- name: postfix 삭제
  dnf:
    name: postfix
    state: absent
```

---

### package 모듈 (범용)
**OS 자동 감지 → 적절한 패키지 매니저 호출**

```yaml
- name: vim, git 설치 (모든 OS)
  package:
    name:
      - vim
      - git
    state: present

- name: Apache 설치 (OS별)
  package:
    name: apache2
  when: ansible_os_family == "Debian"

- package:
    name: httpd
  when: ansible_os_family == "RedHat"
```

---

### pip 모듈 (Python 패키지)
| 파라미터 | 설명 |
|----------|------|
| `name` | 패키지 이름 |
| `virtualenv` | 가상 환경 경로 |
| `requirements` | requirements.txt 파일 |

```yaml
- name: Flask, gunicorn 설치
  pip:
    name:
      - Flask
      - gunicorn
    state: present
    virtualenv: /opt/my_app/venv

- name: requirements.txt로 설치
  pip:
    requirements: /opt/my_app/requirements.txt
    virtualenv: /opt/my_app/venv
```

## 2.3 서비스 관리 모듈

### service 모듈 (범용)
| 파라미터 | 설명 |
|----------|------|
| `name` | 서비스 이름 |
| `state` | started/stopped/restarted/reloaded |
| `enabled` | 부팅 시 자동 시작 |

```yaml
- name: nginx 시작 및 활성화
  service:
    name: nginx
    state: started
    enabled: yes

- name: cron 재시작
  service:
    name: cron
    state: restarted

- name: bluetooth 중지
  service:
    name: bluetooth
    state: stopped
    enabled: no
```

---

### systemd 모듈 (Systemd 전용)
| 추가 파라미터 | 설명 |
|----------------|------|
| `daemon_reload` | 유닛 파일 변경 후 reload |
| `masked` | 서비스 마스킹 (강력 비활성화) |

```yaml
- name: nginx 시작
  systemd:
    name: nginx
    state: started
    enabled: yes

- name: ModemManager 마스킹
  systemd:
    name: ModemManager
    state: stopped
    enabled: no
    masked: yes
```

## 2.4 사용자 및 그룹 관리

### group 모듈
| 파라미터 | 설명 |
|----------|------|
| `name` | 그룹 이름 |
| `state` | present/absent |
| `gid` | GID 지정 |
| `system` | 시스템 그룹 |

```yaml
- name: developers 그룹 생성
  group:
    name: developers
    state: present

- name: admins 그룹 (GID 2000)
  group:
    name: admins
    gid: 2000
    state: present

- name: app_svc 시스템 그룹
  group:
    name: app_svc
    system: yes

- name: old_group 삭제
  group:
    name: old_group
    state: absent
```

---

### user 모듈
| 파라미터 | 설명 |
|----------|------|
| `name` | 사용자 이름 |
| `state` | present/absent/locked |
| `password` | 암호화된 해시 |
| `group` | 주 그룹 |
| `groups` | 보조 그룹 |
| `shell` | 기본 셸 |
| `create_home` | 홈 디렉터리 생성 |

**비밀번호 해시 생성**:
```bash
openssl passwd -6 '1004P@ssw0rd!'
```

```yaml
- name: hong 사용자 생성
  user:
    name: hong
    comment: "홍길동"
    shell: /bin/bash

- name: tomcat 서비스 계정
  user:
    name: tomcat
    shell: /sbin/nologin
    create_home: no
    system: yes

- name: 장길산 계정
  user:
    name: jang
    uid: 2001
    group: developers
    groups: admins,www-data
    password: "{{ new_user_password_hash }}"

- name: bob을 devops 그룹에 추가
  user:
    name: bob
    groups: devops
    append: yes

- name: hong 계정 잠금
  user:
    name: hong
    state: locked

- name: old_user 삭제 (홈 포함)
  user:
    name: old_user
    state: absent
    remove: yes
```

## 종합 실전 예제: 개발팀 환경 구성

```yaml
- name: 신규 서버에 개발팀 환경 구성
  hosts: all_servers
  become: yes
  vars_files:
    - vars/secrets.yml

  tasks:
    - name: developers 그룹 생성
      group:
        name: developers
        state: present

    - name: lead_dev 계정 생성
      user:
        name: lead_dev
        comment: "Lead Developer"
        shell: /bin/bash
        group: developers
        groups: sudo
        append: yes
        password: "{{ new_user_password_hash }}"

    - name: junior_dev 계정 생성
      user:
        name: junior_dev
        shell: /bin/bash
        group: developers
        password: "{{ new_user_password_hash }}"
```
