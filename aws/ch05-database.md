## 5.2 Amazon RDS

### RDS란?
- **Relational Database Service** = 관리형 관계형 DB
- AWS가 설치, 패치, 업그레이드, 장애 조치, 백업 처리

### RDS 생성 과정
1. DB 엔진 선택 (MySQL, PostgreSQL, Aurora 등)
2. 인스턴스 크기 선택
3. DB 이름, 관리자 계정/비밀번호 입력
4. 생성

### 지원 DB 엔진

| 엔진 | 설명 |
|------|------|
| MySQL | 가장 많이 사용 |
| PostgreSQL | 오픈소스 |
| MariaDB | MySQL 기반 |
| Aurora | AWS 전용, MySQL 5배, PostgreSQL 3배 빠름 |

### Multi-AZ 배포
- **보조 인스턴스**를 다른 AZ에 생성
- 데이터 동기화 복제
- 기본 장애 → 보조 자동 승격
- 앱은 **하나의 엔드포인트**만 사용

### RDS 장점
| 장점 | 설명 |
|------|------|
| **운영 부담 최소화** | 설치/패치/백업 자동화 |
| **고가용성** | Multi-AZ 지원 |
| **확장성** | 인스턴스 크기/스토리지 확장 |
| **내구성** | 자동 백업/스냅샷 |

---

## 5.3 관계형 DB vs NoSQL DB

### 비유
| DB | 비유 |
|------|------|
| **관계형 (RDS)** | 잘 정리된 도서관 (JOIN 필요) |
| **NoSQL (DynamoDB)** | 거대한 사물함 (Key-Value) |

### 관계형 DB 문제
- 여러 테이블 JOIN → 트래픽 많을 때 느려짐

### DynamoDB 장점
- Key로 **바로 조회** → JOIN 없음
- 밀리초 단위 응답

---

## 5.4 Amazon DynamoDB

### DynamoDB란?
- **서버리스 NoSQL DB**
- 인프라 관리 필요 없음
- 자동 확장/축소, 고가용성

### 데이터 구조
```
테이블 (Table) → 항목 (Item) → 속성 (Attribute)
```

### RDS vs DynamoDB

| 항목 | RDS | DynamoDB |
|------|-----|----------|
| **스키마** | 엄격 | 유연 |
| **쿼리** | SQL, JOIN | 단일 테이블 중심 |
| **확장성** | 수동 조정 | 자동 확장 |
| **지속성** | 인스턴스 기반 | 서버리스 |
| **비용** | 인스턴스 기준 | 요청/데이터 처리량 |

### DynamoDB 장점
- 자동 확장
- 고가용성 (여러 AZ 복제)
- 밀리초 단위 응답
- 유연한 스키마

### DynamoDB 실습
1. 테이블 생성: `employees`
2. 기본 키: `employee_id`
3. EC2 앱에서 직원 추가 → DynamoDB 저장 확인

## 5.5 DB 서비스 선택

| 요구사항 | DB |
|----------|-----|
| 복잡한 SQL, JOIN | **RDS** |
| 단순 조회/쓰기, 빠른 성능 | **DynamoDB** |
| 대규모, 초당 수백만 요청 | **DynamoDB** |
| 관계 분석 | **RDS** |
