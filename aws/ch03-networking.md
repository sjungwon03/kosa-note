# Chapter 3: AWS 네트워킹

## 3.1 Amazon VPC

### VPC란?
- **Virtual Private Cloud** = 클라우드 데이터 센터의 **가상 경계(벽)**
- 내부 리소스는 기본적으로 외부와 차단

### VPC 구축 과정

| 단계 | 설명 |
|------|------|
| 1. VPC 생성 | 리전 선택, CIDR 블록 설정 (예: 10.1.0.0/16) |
| 2. 서브넷 생성 | VPC 내부 공간 쪼개기 |
| 3. IGW 연결 | 인터넷 게이트웨이 생성 후 VPC에 Attach |
| 4. 라우팅 테이블 | 퍼블릭/프라이빗 서브넷 연결 |

### 서브넷 비유
```
VPC = 도시
서브넷 = 동네 (강남동, 서초동, 마포동)
```

### 서브넷 종류

| 종류 | 설명 | 용도 |
|------|------|------|
| **퍼블릭 서브넷** | 외부 인터넷 연결 가능 | 웹 서버 |
| **프라이빗 서브넷** | 외부 직접 노출 안 됨 | DB |

### 고가용성 (HA)
- **최소 2개의 AZ**에 동일 구조 배치
- 하나의 AZ 다운 → 다른 AZ에서 서비스 유지

## 3.2 인터넷 게이트웨이 (IGW)

### IGW란?
- VPC를 **인터넷에 연결**하는 장치
- 모뎀이 집을 인터넷에 연결하는 것과 같은 역할

### 가상 프라이빗 게이트웨이 (VGW)
- 인터넷 대신 **온프레미스 ↔ AWS VPC** 연결
- VPN으로 암호화된 프라이빗 연결

---

## 3.3 VPC 라우팅

### 라우팅 테이블

| 라우팅 테이블 | 경로 | 연결 서브넷 |
|--------------|------|-------------|
| **Main RT** | 10.1.0.0/16 → Local | 프라이빗 서브넷 |
| **Public RT** | 10.1.0.0/16 → Local<br>0.0.0.0/0 → IGW | 퍼블릭 서브넷 |

### 트래픽 흐름
```
사용자 → 인터넷 → IGW → 퍼블릭 서브넷 (EC2)
          ↓
        응답 → IGW → 사용자
```

### 프라이빗 서브넷
- IGW 라우팅 없음 → **인터넷 직접 접근 불가** (보안 강화)

## 3.4 VPC 보안

### 네트워크 ACL (NACL)

| 특징 | 설명 |
|------|------|
| **단위** | 서브넷 단위 방화벽 |
| **기본값** | 모든 인바운드/아웃바운드 허용 |
| **Stateless** | 요청과 응답 따로 규칙 추가 필요 |

### 보안 그룹 (Security Group)

| 특징 | 설명 |
|------|------|
| **단위** | EC2 인스턴스 단위 방화벽 |
| **기본값** | 인바운드 차단, 아웃바운드 전부 허용 |
| **Stateful** | 응답 트래픽 자동 허용 |

### Stateful 비유
> **기억력 좋은 경비원**: 나간 요청에 대한 응답은 자동 통과

### Stateless 비유
> **기억력 없는 경비원**: 들어오고 나가는 트래픽 모두 규칙 필요

### 보안 그룹 예시 (웹 서버)
- 인바운드: Allow TCP 80, 443 (0.0.0.0/0)
- 아웃바운드: 기본 허용
