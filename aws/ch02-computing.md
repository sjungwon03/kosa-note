## 2.2 주요 컴퓨팅 서비스

| 서비스 | 설명 | 용도 |
|--------|------|------|
| **EC2** | 가상 머신 (VM) | 웹 서버, DB 서버 |
| **ECS/EKS** | 컨테이너 오케스트레이션 | 마이크로서비스 |
| **Lambda** | 서버리스 컴퓨팅 | 이벤트 기반 작업 |

### 선택 기준
| 목적 | 서비스 |
|------|--------|
| 전통적 앱, VM 필요 | **EC2** |
| 마이크로서비스, 이식성 | **컨테이너 (ECS/EKS)** |
| 이벤트 기반, 짧은 작업 | **Lambda** |

## 2.3 Amazon EC2

### EC2 장점
| 장점 | 설명 |
|------|------|
| **유연성** | 몇 분 안에 생성/종료 |
| **비용 효율성** | 초/시간 단위 과금 |
| **확장성** | 중지/종료 시 비용 없음 |
| **OS 선택** | Linux, Ubuntu, Windows, MacOS |

### AMI (Amazon Machine Image)
- EC2 실행용 **기본 설계도**
- 포함: OS, 사전 설치 소프트웨어, 초기 설정
- 종류: AWS 제공, 커뮤니티, 커스텀 AMI

### 인스턴스 유형 (패밀리)

| 유형 | 특징 | 용도 |
|------|------|------|
| **범용 (M)** | CPU/메모리/스토리지 균형 | 웹 서버, 일반 앱 |
| **컴퓨팅 최적화 (C)** | CPU 성능 중요 | 고성능 연산 |
| **메모리 최적화 (R)** | 대용량 메모리 | 인메모리 DB |
| **스토리지 최적화 (I)** | 대용량 디스크 I/O | 빅데이터, 로그 |
| **GPU 최적화 (G)** | 그래픽/AI/영상 | AI, 영상 처리 |

### 인스턴스 크기
```
nano → micro → small → medium → large → xlarge
예: t3.micro, m5.large, g4dn.xlarge
```

## 2.4 EC2 인스턴스 수명 주기

### 주요 상태

| 상태 | 설명 | 과금 |
|------|------|------|
| **Pending** | 부팅 준비 중 | - |
| **Running** | 정상 작동 | ✅ 요금 청구 |
| **Reboot** | 재부팅 | ✅ |
| **Stopped** | 전원 끈 상태, 데이터 유지 | ❌ (EBS 과금만) |
| **Hibernate** | 메모리 상태 저장, 빠른 재개 | ❌ |
| **Terminated** | 완전 삭제 | ❌ |

### 비용 관점
- **실행 중일 때만 요금 부과**
- 중지 상태 → 인스턴스 요금 없음 (스토리지 요금만)

### 운영 팁
- 종료 방지 (Termination Protection) 설정
- 근무 시간만 실행 → 비용 절약

## 2.5 컨테이너 서비스

### 컨테이너란?
- 앱 실행용 **코드 + 라이브러리 + 설정** 패키지
- 특징: 이식성, 빠른 부팅, 자원 효율

### AWS 컨테이너 서비스

| 서비스 | 설명 |
|--------|------|
| **ECS** | AWS 네이티브 컨테이너 오케스트레이션 |
| **EKS** | Kubernetes 기반 오케스트레이션 |
| **Fargate** | 서버리스 컨테이너 실행 (호스트 관리 불필요) |

### 활용 사례
- 웹 앱 호스팅
- 마이크로서비스
- 리프트 앤 시프트 마이그레이션
- ML 모델 서빙

---

## 2.6 서버리스 컴퓨팅

### EC2/컨테이너 vs 서버리스

| 특징 | EC2/컨테이너 | 서버리스 |
|------|-------------|----------|
| 인프라 관리 | 고객 책임 | AWS 책임 |
| 운영 부담 | 있음 | 없음 |
| 제어권 | 많음 | 제한 |

### 공동 책임 모델 변화
- **EC2**: OS 패치, 보안 업데이트 = 고객 책임
- **서버리스**: OS/플랫폼 패치 = AWS 책임
- **공통 고객 책임**: 데이터 암호화, IAM, 앱 코드 보안

## 2.7 AWS Lambda

### Lambda란?
- **서버리스 컴퓨팅 서비스**
- 코드만 올리면 **이벤트 발생 시 자동 실행**
- 밀리초 단위 과금

### 실행 흐름
```
코드 업로드 → 트리거 설정 → 이벤트 발생 → Lambda 자동 실행
```

### 특징
- 다양한 언어 지원 (Python, Node.js, Java, Go)
- 실행 시간 제한: **최대 15분**
- 짧고 빠른 작업에 적합

### S3 이미지 리사이즈 예시
```
1. S3 input 폴더에 이미지 업로드
2. S3 이벤트 → Lambda 트리거
3. Lambda가 이미지 리사이즈
4. 결과 → S3 output 폴더 저장
```

| 요구사항 | 서비스 |
|----------|--------|
