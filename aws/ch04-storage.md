# Chapter 4: AWS 스토리지

## 4.1 스토리지 유형

### 3가지 스토리지 유형

| 유형 | 설명 | AWS 서비스 | 용도 |
|------|------|-----------|------|
| **파일 스토리지** | 폴더/파일 계층 구조 | EFS | 네트워크 드라이브, 공유 |
| **블록 스토리지** | 고정 크기 블록 단위 | EBS | 빠른 업데이트, 트랜잭션 |
| **객체 스토리지** | 파일 전체를 객체로 저장 | S3 | 대용량 정적 파일 |

### 직원 디렉터리 앱 매핑

| 데이터 | 스토리지 |
|--------|----------|
| 운영체제/앱 파일 | **EBS** (블록 스토리지) |
| 직원 사진 | **S3** (객체 스토리지) |
| 직원 데이터 | **RDS** (데이터베이스) |

## 4.2 Amazon EBS (블록 스토리지)

### EBS란?
- EC2와 **네트워크로 연결**되는 블록 스토리지
- **영구성 (Persistent)**: EC2 종료 후 데이터 보존

### 인스턴스 스토어 vs EBS

| 특징 | 인스턴스 스토어 | EBS |
|------|----------------|-----|
| **속도** | 매우 빠름 (로컬) | 네트워크 지연 있음 |
| **영구성** | 휘발성 (인스턴스 종료 시 소멸) | 영구 저장 |
| **이동성** | 해당 인스턴스만 | 다른 인스턴스로 연결 가능 |
| **용도** | 캐시, 임시 데이터 | OS, DB, 앱 데이터 |

### EBS 볼륨 타입

| 타입 | 특징 | 용도 |
|------|------|------|
| **gp3** | 범용 SSD | 대부분 워크로드 |
| **io2** | 고성능 IOPS | DB, 트랜잭션 |
| **st1** | 처리량 최적화 HDD | 로그, 빅데이터 |
| **sc1** | 콜드 HDD | 자주 접근하지 않는 데이터 |

### EBS 스냅샷
- **증분 백업**: 변경된 블록만 저장
- S3에 저장, **내구성 99.999999999% (11 9's)**

## 4.3 Amazon S3 (객체 스토리지)

### S3란?
- **인터넷 앨범** 같은 무한 저장소
- URL로 접근, 언제 어디서든 가능
- 기본 비공개 → 접근 권한 지정 필요

### S3 vs EBS 비교

| 항목 | EBS | S3 |
|------|-----|-----|
| 저장 방식 | 외장 하드 | 인터넷 앨범 |
| 적합 데이터 | OS, 프로그램, 자주 바뀌는 데이터 | 사진, 이미지, 문서 |
| 공유 | 한 인스턴스만 | URL로 누구나 접근 가능 |
| 확장성 | 제한적 | 무한대 |
| 보안 | 인스턴스 내부 | 기본 비공개, 정책 제어 |

### S3 버킷 생성
1. 버킷 이름 지정
2. 리전 선택
3. 접근 권한 설정 (공개/비공개)
4. 객체 업로드

---

## 4.4 Amazon EFS (파일 스토리지)

### EFS란?
- **네트워크 파일 시스템**
- 여러 EC2 인스턴스가 **동시에 접근 가능**
- Linux 기반

### 용도
- 공유 파일 시스템
- 웹 서버 간 파일 공유
- 개발/테스트 환경

---

## 4.5 스토리지 선택 가이드

| 요구사항 | 스토리지 |
|----------|----------|
| OS/앱 실행, 빠른 I/O | **EBS** |
| 이미지/문서, 정적 파일 | **S3** |
| 여러 서버 파일 공유 | **EFS** |
| 임시/캐시 데이터 | **인스턴스 스토어** |
