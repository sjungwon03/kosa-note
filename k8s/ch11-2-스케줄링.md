# Chapter 11-2: 스케줄링

## 1. 파드 생성 과정

### 1.1 전체 흐름

```
1. User → kubectl apply
2. API Server → 검증 → etcd 저장 (Pending)
3. Scheduler → 노드 선택 → 바인딩
4. Kubelet → 파드 감지 → 컨테이너 실행
5. Kubelet → 상태 보고 → API Server → etcd
```

### 1.2 단계별 설명

| 단계              | 담당         | 설명                             |
| ----------------- | ------------ | -------------------------------- |
| 1. 파드 생성 요청 | User/kubectl | YAML 제출                        |
| 2. API 서버 수신  | API Server   | 유효성 검사 → etcd 저장          |
| 3. 스케줄링       | Scheduler    | 노드 선택 → 바인딩               |
| 4. 파드 실행      | Kubelet      | CRI/CNI/CSI 호출 → 컨테이너 실행 |
| 5. 상태 보고      | Kubelet      | Running/Ready 상태 보고          |

### 1.3 스케줄러 동작

```
필터링 (Filtering) → 조건에 맞는 노드 목록
스코어링 (Scoring) → 가장 적합한 노드 선택
바인딩 (Binding) → nodeName 업데이트
```

### 1.4 Kubelet 작업

```
CRI → 컨테이너 런타임 호출 (이미지 pull)
CNI → 네트워크 설정 (IP 할당)
CSI → 스토리지 설정 (볼륨 연결)
```

## 2. Node Label

### 2.1 정의

- 노드에 붙이는 키-값 쌍
- 스케줄링 조건으로 사용

### 2.2 주요 명령어

```bash
# 레이블 조회
kubectl get nodes --show-labels

# 레이블 추가
kubectl label node node1 disktype=ssd

# 레이블 삭제
kubectl label node node1 disktype-
```

### 2.3 nodeSelector 사용

```yaml
apiVersion: v1
kind: Pod
metadata:
  name: nginx-pod
spec:
  nodeSelector:
    disktype: ssd
  containers:
    - name: nginx
      image: nginx
```

## 3. Node Affinity

### 3.1 정의

- nodeSelector보다 **더 강력하고 유연한** 노드 선택 조건

### 3.2 종류

| 종류                                                | 설명                         |
| --------------------------------------------------- | ---------------------------- |
| **requiredDuringSchedulingIgnoredDuringExecution**  | 조건 미충족 → 스케줄링 안 됨 |
| **preferredDuringSchedulingIgnoredDuringExecution** | 조건 미충족 → 다른 노드 가능 |

### 3.3 예제

```yaml
apiVersion: v1
kind: Pod
metadata:
  name: nginx-pod
spec:
  affinity:
    nodeAffinity:
      requiredDuringSchedulingIgnoredDuringExecution:
        nodeSelectorTerms:
          - matchExpressions:
              - key: disktype
                operator: In
                values:
                  - ssd
                  - nvme
      preferredDuringSchedulingIgnoredDuringExecution:
        - weight: 80
          preference:
            matchExpressions:
              - key: zone
                operator: In
                values:
                  - zone-a
```

## 4. Pod Affinity/Anti-Affinity

### 4.1 정의

- 특정 파드가 실행되는 노드에 **함께 실행** 또는 **실행**

### 4.2 종류

| Affinity         | Anti-Affinity    |
| ---------------- | ---------------- |
| 함께 실행        | 실행 피하기      |
| 같은 노드에 배치 | 다른 노드에 배치 |

### 4.3 예제

```yaml
apiVersion: v1
kind: Pod
metadata:
  name: web-pod
spec:
  affinity:
    podAffinity:
      requiredDuringSchedulingIgnoredDuringExecution:
        - labelSelector:
            matchLabels:
              app: cache
          topologyKey: kubernetes.io/hostname
    podAntiAffinity:
      preferredDuringSchedulingIgnoredDuringExecution:
        - weight: 100
          podAffinityTerm:
            labelSelector:
              matchLabels:
                app: web
            topologyKey: kubernetes.io/hostname
```

## 5. Taints & Tolerations

### 5.1 정의

| Taints      | Tolerations |
| ----------- | ----------- |
| 노드에 적용 | 파드에 적용 |
| "밀어내기"  | "허용하기"  |

### 5.2 Taints Effect

| Effect               | 설명                                         |
| -------------------- | -------------------------------------------- |
| **NoSchedule**       | 새 파드 스케줄링 안 됨 (기존 파드 유지)      |
| **PreferNoSchedule** | 스케줄링 피하려고 시도 (가능하면 배치 안 함) |
| **NoExecute**        | 새 파드 스케줄링 안 됨 + 기존 파드 축출      |

### 5.3 Taints 명령어

```bash
# 테인트 추가
kubectl taint nodes node1 dedicated=team-a:NoSchedule

# 테인트 삭제
kubectl taint nodes node1 dedicated:NoSchedule-
```

### 5.4 Toleration 예제

```yaml
apiVersion: v1
kind: Pod
metadata:
  name: my-team-a-app
spec:
  containers:
    - name: my-app
      image: nginx
  tolerations:
    - key: "dedicated"
      operator: "Equal"
      value: "team-a"
      effect: "NoSchedule"
```

### 5.5 tolerationSeconds

- NoExecute 효과에서 축출 유예 시간

```yaml
tolerations:
  - key: "gpu"
    operator: "Equal"
    value: "nvidia"
    effect: "NoExecute"
    tolerationSeconds: 300 # 300초 유예
```

## 6. nodeName

### 6.1 정의

- 특정 노드에 **강제 스케줄링**
- 스케줄러를 거치지 않음

### 6.2 예제

```yaml
apiVersion: v1
kind: Pod
metadata:
  name: nginx-pod
spec:
  nodeName: node1
  containers:
    - name: nginx
      image: nginx
```

## 7. 사용 사례

| 기법               | 사용 사례                  |
| ------------------ | -------------------------- |
| nodeSelector       | 단순 노드 선택 (GPU 노드)  |
| Node Affinity      | 복잡한 조건 (AND, OR, NOT) |
| Pod Affinity       | 함께 실행 (Web + Cache)    |
| Pod Anti-Affinity  | 분산 실행 (HA 구성)        |
| Taints/Tolerations | 전용 노드, 특수 하드웨어   |
| nodeName           | 테스트, 강제 배치          |
