# Chapter 14: 모니터링 (Prometheus, Grafana, Helm)

## 1. Prometheus

### 1.1 정의
- 오픈 소스 **모니터링 및 알림 시스템**
- CNCF 프로젝트, K8s와 함께 가장 중요한 프로젝트

### 1.2 핵심 특징

| 특징 | 설명 |
|------|------|
| **시계열 데이터** | 메트릭 이름 + 레이블 + 타임스탬프 + 값 |
| **Pull 기반 수집** | Prometheus가 대상에 HTTP 요청 |
| **PromQL** | 강력한 쿼리 언어 |
| **서비스 디스커버리** | 동적 환경에서 자동 대상 찾기 |
| **알림** | Alertmanager로 이메일/Slack 알림 |

### 1.3 Pull vs Push
| Pull (Prometheus) | Push (다른 시스템) |
|-------------------|-------------------|
| Prometheus가 가져옴 | 대상이 보냄 |
| 중앙 관리 가능 | 대상이 주도 |
| 장애 감지 쉬움 | 장애 감지 어려움 |

### 1.4 핵심 구성
| 구성 | 설명 |
|------|------|
| **Target** | 메트릭 수집 대상 (`ip:port/metrics`) |
| **Exporter** | 메트릭 변환 에이전트 (node_exporter) |
| **Alertmanager** | 알림 라우팅 (Email, Slack, PagerDuty) |

### 1.5 시계열 데이터 구조
```
메트릭 이름: http_requests_total
레이블: method="POST", path="/api/users"
타임스탬프: 2025-05-29T10:00:00Z
값: 1024
```

### 1.6 PromQL 예시
```promql
# 지난 5분간 초당 평균 HTTP 요청
rate(http_requests_total[5m])

# CPU 사용량 80% 초과 노드
node_cpu_usage > 0.8

# 내년 이맘때 예상 디스크 사용량
predict_linear(node_disk_usage[1h], 365*24*60*60)
```

## 2. Grafana

### 2.1 정의
- Prometheus 데이터 **시각화 대시보드**
- 다양한 그래프, 차트, 알림

### 2.2 특징
- 다중 데이터 소스 (Prometheus, InfluxDB, MySQL)
- 커스텀 대시보드
- 알림 기능

---

## 3. cAdvisor

### 3.1 정의
- 컨테이너 리소스 사용량 **실시간 수집**
- CPU, 메모리, 파일시스템, 네트워크

### 3.2 특징
- K8s에 내장 (kubelet에 통합)
- `/metrics/cadvisor` 엔드포인트

---

## 4. Metrics Server

### 4.1 정의
- K8s 내장 메트릭 API 제공
- HPA, VPA에서 사용

### 4.2 설치
```bash
kubectl apply -f https://github.com/kubernetes-sigs/metrics-server/releases/latest/download/components.yaml

# HTTPS 문제 해결
kubectl edit deployment metrics-server -n kube-system
# args에 --kubelet-insecure-tls 추가
```

### 4.3 확인
```bash
kubectl top nodes
kubectl top pods
```

## 5. Helm

### 5.1 정의
- K8s **패키지 매니저**
- Chart (템플릿)로 앱 배포

### 5.2 개념
| 개념 | 설명 |
|------|------|
| **Chart** | 앱 정의 (템플릿) |
| **Release** | Chart 실행 인스턴스 |
| **Repository** | Chart 저장소 |

### 5.3 명령어
```bash
helm repo add prometheus-community https://prometheus-community.github.io/helm-charts
helm repo update
helm install prometheus prometheus-community/prometheus
helm list
helm upgrade prometheus prometheus-community/prometheus
helm uninstall prometheus
```

## 6. ArgoCD

### 6.1 정의
- **GitOps** 기반 CD 도구
- Git 저장소 → K8s 자동 배포

### 6.2 특징
- 선언적 배포
- 자동 동기화
- 롤백 기능

### 6.3 구성
```
Git 저장소 → ArgoCD → K8s 클러스터
```

## 7. Terraform

### 7.1 정의
- **IaC (Infrastructure as Code)** 도구
- 인프라를 코드로 정의하고 관리

### 7.2 특징
- 멀티 클라우드 (AWS, GCP, Azure)
- 선언적 구성
- 상태 관리

### 7.3 K8s와 함께 사용
- 클러스터 생성 (EKS, GKE, AKS)
- 노드, 네트워크, 스토리지 프로비저닝

## 8. Ansible

### 8.1 정의
- **구성 관리** 도구
- 서버/앱 설정 자동화

### 8.2 특징

## 9. Prometheus Stack 배포

### 9.1 Helm로 배포
```bash
helm repo add prometheus-community https://prometheus-community.github.io/helm-charts
helm install prometheus-stack prometheus-community/kube-prometheus-stack
```

### 9.2 포함 구성
- Prometheus Server
- Alertmanager
- Grafana
- Node Exporter
- kube-state-metrics
