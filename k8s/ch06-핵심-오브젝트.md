# Chapter 6: 핵심 오브젝트

## 1. 전체 구조 (3계층)

```
[접근 레이어]   Service / Ingress     ← 네트워크 접근
[관리 레이어]   Deployment / ReplicaSet ← 배포와 운영
[실행 레이어]   Pod / Container        ← 실제 실행
```

---

## 2. Pod

### 2.1 정의
- 쿠버네티스에서 **가장 작은 실행 단위**
- 하나 이상의 컨테이너를 포함

### 2.2 핵심 특징
| 특징 | 설명 |
|------|------|
| 고유 IP | Pod마다 고유 IP 할당 |
| localhost 통신 | 내부 컨테이너는 localhost로 통신 |
| Volume 공유 | 내부 컨테이너 간 스토리지 공유 |
| 휘발성 | 재생성 시 IP 변경 |

### 2.3 내부 구조
```
Pod
 ├─ Network Namespace (IP 1개)
 ├─ Volume
 ├─ Container (App)
 └─ Container (Sidecar)
```

### 2.4 한계
- 자동 복구 불가
- 스케일링 불가
- **직접 운영 비권장**

### 2.5 YAML 예시
```yaml
apiVersion: v1
kind: Pod
metadata:
  name: nginx-pod-example
  labels:
    app: webserver
spec:
  containers:
  - name: nginx-container
    image: nginx:latest
    ports:
    - containerPort: 80
```

## 3. ReplicaSet

### 3.1 정의
- **지정한 개수의 Pod를 항상 유지**하는 컨트롤러

### 3.2 동작 원리
```
Pod 개수 부족 → 새 Pod 생성
Pod 초과     → Pod 삭제
```

### 3.3 한계
- 롤링 업데이트 불가
- 롤백 불가
- 버전 관리 불가

> **실무에서 ReplicaSet을 직접 사용하지 않음**

---

## 4. Deployment

### 4.1 정의
- ReplicaSet을 관리하는 상위 컨트롤러
- **실무 운영의 핵심**

### 4.2 제공 기능
| 기능 | 설명 |
|------|------|
| 자동 복구 | Pod 실패 시 자동 재생성 |
| 스케일링 | Pod 수 조절 |
| 롤링 업데이트 | 무중단 버전 업데이트 |
| 롤백 | 이전 버전으로 복구 |

### 4.3 내부 구조
```
Deployment
 ├─ ReplicaSet v1 (이전 버전)
 ├─ ReplicaSet v2 (현재 버전)
 └─ ReplicaSet v3 (새 버전)
```

### 4.4 무중단 배포 과정
```
1. 새 ReplicaSet 생성
2. Pod를 순차적으로 교체 (Rolling Update)
3. 서비스 중단 없음
```

---

## 5. Service

### 5.1 정의
- Pod 앞단에 위치한 **고정 네트워크 진입점**

### 5.2 역할
| 역할 | 설명 |
|------|------|
| 고정 IP 및 DNS | Pod IP는 변하지 Service는 고정 |
| 로드밸런싱 | 요청을 여러 Pod로 분산 |
| 안정적 접근 | Pod 변경과 무관하게 접근 |

### 5.3 Service 타입
| 타입 | 설명 | 사용 시나리오 |
|------|------|---------------|
| **ClusterIP** | 내부 통신용 (기본) | 내부 서비스 간 통신 |
| **NodePort** | 외부 테스트용 | 노드 포트로 외부 접근 |
| **LoadBalancer** | 클라우드 환경용 | 외부 로드밸런서 할당 |

---

## 6. External Service

### 6.1 ExternalName Service
- 외부 DNS를 K8s 서비스처럼 사용
```yaml
type: ExternalName
externalName: db.example.com
```

### 6.2 Endpoint 기반 외부 연동
- 외부 VM, 레거시 시스템 연동
- IP 직접 지정

---

## 7. 전체 동작 흐름

```
사용자 요청
    ↓
Service (로드밸런싱)
    ↓
Deployment (배포 관리)
    ↓
ReplicaSet (Pod 수 유지)
    ↓
Pod (실행)
    ↓
Container (앱)
```

---

## 8. 요약

| 오브젝트 | 역할 |
|----------|------|
| Pod | 실행 단위 (최소) |
| ReplicaSet | Pod 수 유지 |
| Deployment | 배포 및 운영 (**실무 핵심**) |
| Service | 네트워크 접근 |
| External Service | 외부 시스템 연동 |
